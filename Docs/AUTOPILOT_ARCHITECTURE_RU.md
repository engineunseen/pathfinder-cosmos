# Спецификация Режима "Автопилот v0.9": Тактический Контур Управления

> **Статус:** ОПЕРАЦИОННЫЙ (Operational) - см. `AUTOPILOT_STATUS_RU.md`
> **Версия:** v0.9.45
> **ВНИМАНИЕ:** Использование `gemini-1.5` запрещено. Только `gemini-3`.

## 1. Архитектура Агента "Автопилот"

"Автопилот" (или Тактический Гид) — это интеллектуальный агент уровня 2 (L2), который отвечает за непосредственное управление марсоходом в реальном времени. В отличие от стратегического архитектора, который планирует "глобальный" маршрут, автопилот принимает решения "здесь и сейчас", основываясь на мгновенной телеметрии и сложной физической модели.

### 1.1 Входные данные (Контекст агента)
Агент получает следующий набор данных при каждом такте (update loop) в промпте (`src/aiNavigator.js:getAutopilotCommand`):

1.  **Карта Местности (LIDAR TOPO SWEEP):**
    *   Агент не "видит" картинку целиком, как архитектор. Вместо этого он получает данные от виртуального Лидара.
    *   **Механизм:** Функция `getLidarData` выполняет серию математических лучей (Raycast) в 8 направлениях (0°, 45°, 90°...) на дистанции 5 и 10 метров от центра ровера.
    *   **Данные:** Возвращает текстовый отчет о перепаде высот. Например: `- 90°: 5m:[+0.5m] 10m:[-1.2m]`. Это позволяет агенту "нащупывать" рельеф вокруг себя.

2.  **Глобальный Маршрут (MISSION ROUTE):**
    *   Список ближайших точек сплайна, построенного стратегическим архитектором.
    *   **Задача:** Автопилот обязан следовать этому пути, но имеет право отклоняться для избежания локальных опасностей (камни, ямы), с обязательным возвращением на маршрут.

3.  **Кинетическая Телеметрия (TELEMETRY CONTEXT):**
    *   **POSITION:** Текущие [X, Z] координаты.
    *   **VELOCITY:** Вектор скорости.
    *   **ANGULAR VELOCITY (v0.9.41):** Скорость вращения вокруг осей. Позволяет рассчитывать будущую ориентацию.

4.  **Сенсорный Слой Монте-Карло (KINETIC METRICS):**
    *   **sCVaR / SMaR:** Оценка рисков на основе 60 параллельных симуляций путей.
    *   **DUAL-STREAM TRANSFORMATION (v0.9.42):** Система разделения пеленга. Агент получает **Predicted Bearing** (предсказание на время RTT), в то время как локальные подсистемы используют **Real-time Bearing**.

5.  **Темпоральный Слой (TEMPORAL SIGHT v0.9.44):**
    *   **ADAPTIVE FORESIGHT:** Автоматическое сокращение горизонта прогноза при сближении с целью. Это устраняет ложные команды разворота ("пируэты") у самого маяка.

### 1.2 Логика Принятия Решений (Operational Directive)

Агент должен следовать строгой логике, зашитой в промпт:

1.  **Фазовый Анализ (Phase Analysis):**
    *   *Инерциальная фаза (Полет):* Если колес на земле < 3. Руление бесполезно. Агент должен готовиться к удару, минимизируя вращение.
    *   *Кинетическая фаза (Езда):* Если колес >= 3. Активное управление тягой и рулем.

2.  **Приоритет Миссии (Mission Priority):**
    *   Главная цель: Достичь `CURRENT WAYPOINT`.
    *   Второстепенная цель: Оставаться в пределах 5м от `MISSION ROUTE`.

3.  **Метрический Контроль (Metric-Driven Control):**
    *   Данные Монте-Карло (sCVaR, SMaR) имеют приоритет над скоростью.
    *   "Тише едешь — дальше будешь". Агент должен "бояться" высоких значений sCVaR.

### 1.3 Выходные данные (JSON)

Агент обязан вернуть СТРОГИЙ JSON объект:
```json
{
  "steer": -0.5,   // Руль: от -1.0 (влево) до 1.0 (вправо)
  "throttle": 0.3, // Газ/Тормоз: от -1.0 (назад) до 1.0 (вперед)
  "reasoning": "High sCVaR detected. Reducing speed to regain traction." // Объяснение действий
}
```

## 2. Текущие Проблемы и Решения (v0.9.6)

### Проблема Парсинга ("PARSING_ERROR")
Из-за использования экспериментальных моделей `gemini-3-flash-preview` (так как 1.5 запрещена), агент иногда склонен к "болтливости", добавляя Markdown (`json`) или вступительный текст. Это ломает простой `JSON.parse`.

**Решение в v0.9.6:**
Внедрен "Robust Brace-Counting Parser" (`extractJson`):
1.  Игнорирует любой текст до первой `{`.
2.  Считает вложенность фигурных скобок, чтобы найти точный конец JSON-объекта `}`.
3.  Игнорирует скобки внутри строковых значений.
4.  Это позволяет извлекать валидный JSON даже из "грзного" ответа.

## 3. Методология Калибровки (Temporal Calibration)

Для достижения стабильности была использована стратегия **"Плоской Скалы" (Seed 999)**:
1.  **Изоляция переменных:** Отключение рельефа и препятствий для проверки чистой математики навигации при задержке > 5с.
2.  **Выявление "Limit Cycles":** Обнаружение спиралевидного поведения и его устранение через Temporal Sight.
3.  **Индукция Характеристик:** Внедрение в "сознание" ИИ понимания массы (150кг) и инерции лунохода, что превратило его из "игрока в GTA" в профессионального оператора научной техники.
